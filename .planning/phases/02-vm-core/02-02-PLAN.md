  <action>
    Create `crates/haydn-vm/src/loop_machine.rs` containing the loop state machine logic. This module provides functions that `lib.rs`'s `step()` calls when encountering LoopStart or LoopEnd opcodes, and manages the recording/replay/skip state transitions.

    **Constants:**
    - `pub(crate) const MAX_LOOP_DEPTH: usize = 256;`

    **Core functions (operating on HaydnVm fields — take &mut references to the relevant fields or take &mut HaydnVm):**

    1. `handle_loop_start(vm: &mut HaydnVm) -> (Operation, Option<EdgeCase>)`:
       - Peek stack top: `vm.stack.last().copied().unwrap_or(0)`
       - If zero or empty: Enter Skipping state.
         - If currently Normal → `LoopState::Skipping(0)`
         - If currently Recording → need to track skip within recording context. Use a skip depth counter on the current LoopFrame or use LoopState::Skipping with a "return-to" state. **Implementation choice:** Add `previous_state: Box<LoopState>` to Skipping variant, or use a separate skip counter. Simpler: track `skip_depth: usize` on HaydnVm directly. When skip_depth > 0, all events are discarded except LoopStart (increment) and LoopEnd (decrement; if reaches 0, exit skip).
         - Return `(Operation::LoopSkipped, None)` or `(Operation::LoopSkipped, Some(EdgeCase::EmptyStackDefault))` if stack was empty.
       - If nonzero: Check nesting depth. If `vm.loop_stack.len() >= MAX_LOOP_DEPTH` → return `(Operation::Noop, Some(EdgeCase::MaxLoopDepthExceeded))`.
         - Push new LoopFrame with empty buffer onto loop_stack.
         - Set loop_state to Recording.
         - Return `(Operation::LoopEntered, None)`.

    2. `handle_loop_end(vm: &mut HaydnVm) -> (Operation, Option<EdgeCase>)`:
       - If loop_state is Normal (unmatched) → return `(Operation::Noop, Some(EdgeCase::UnmatchedLoopEnd))`.
       - If loop_state is Recording:
         - Pop stack top: `vm.stack.pop().unwrap_or(0)`.
         - If nonzero: Pop LoopFrame from loop_stack, convert to ReplayFrame at position 0, push onto replay_stack, set loop_state to Replaying. Return `(Operation::LoopReplaying, None)`.
         - If zero: Discard top LoopFrame. If loop_stack still has frames → stay Recording. Else → Normal. Return `(Operation::LoopExited, None)`.

    3. **Recording behavior:** When loop_state is Recording and a non-LoopEnd event arrives (Push or other Op), the event is BOTH executed AND appended to the top LoopFrame's buffer. Modify `step()` in lib.rs to check loop_state after execution and append the event to the recording buffer.

    4. **Replay behavior:** When loop_state is Replaying, `step()` should NOT dequeue from event_queue. Instead, read the next event from the top ReplayFrame's buffer at its current position. Execute it. Advance position. If position reaches end of buffer:
       - Pop stack top (or 0): if nonzero → reset position to 0 (replay again), return `EndOfBufferReplay`. If zero → discard ReplayFrame; if replay_stack has more → stay Replaying; if loop_stack has frames → Recording (returning to outer recording); else → Normal. Return `EndOfBufferExit`.

    5. **Skip behavior:** Track skip with a `skip_depth: usize` field on HaydnVm (initially 0). When skip_depth > 0:
       - LoopStart event → increment skip_depth
       - LoopEnd event → decrement skip_depth. If reaches 0 → return to previous state (Normal or Recording, based on whether loop_stack is non-empty).
       - All other events → discard (Noop)

    6. **Event queuing during replay:** In `feed()`, if loop_state is Replaying OR skip_depth > 0, push to event_queue. In step(), process replay first; when replay/skip completes, drain event_queue.

    7. `pub fn close(&mut self)` on HaydnVm — discard all loop_stack and replay_stack frames, set loop_state to Normal, set skip_depth to 0. (Spec §5.4 rule 8: stream end.)

    **Update lib.rs step() dispatch:**
    - When processing LoopStart: call `handle_loop_start`
    - When processing LoopEnd: call `handle_loop_end`
    - After executing any event in Recording state: append event to top LoopFrame buffer (EXCEPT LoopEnd itself — spec says loop_end is NOT recorded)
    - In Replaying state: read from replay buffer instead of event_queue
    - In Skipping state: handle skip counting

    **Nested loops during replay:** When replaying and encounter LoopStart with nonzero top → push new LoopFrame, switch to Recording (now recording within a replay context). The replay position is effectively paused. When this nested recording/replay completes, resume the outer replay.

    **Spec §5.4 Rule 1:** loop_start PEEKS (does not pop), loop_end POPS. Critical: do not mix these up.

    **Spec §5.4 Rule 3:** During recording, opcodes are BOTH executed AND recorded. The first pass through a loop body is live execution that also records.

    **Spec §5.4 Rule 4:** Live input queued during replay. feed() → event_queue when replaying.

    Write comprehensive tests in `#[cfg(test)] mod tests` covering all behaviors listed above.
