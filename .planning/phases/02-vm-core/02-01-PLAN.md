From docs/SPECIFICATION.md §3.4 (canonical opcode set):
```
Opcode = dup | swap | drop | rotate
       | add | sub | mul | div | mod
       | eq | gt | lt
       | loop_start | loop_end
       | print_num | print_char | read
       | store | load
```

From docs/SPECIFICATION.md §4.2 (event types):
```
Event = Push(value: i64) | Op(opcode: Opcode)
```

From .planning/phases/02-vm-core/02-CONTEXT.md (D-01, D-04, D-07, D-08):
- step() processes one event, returns StepResult with operation, stack snapshot, optional output, optional edge case
- process_event() convenience wrapper returns Vec<StepResult>
- Output bytes in StepResult as Option field
- Input via provide_input(bytes) feeding VecDeque<u8>, read pops from front or returns 0

From 02-RESEARCH.md §1.2 (chosen API):
- feed(event) → queues live event
- step() → Option<StepResult> (processes one pending event)
- process_event(event) → Vec<StepResult> (feed + drain)

  <action>
    Create `crates/haydn-vm/src/opcodes.rs` with opcode execution functions. Each function takes `&mut HaydnVm` (or the relevant VM fields) and returns `(Operation, Option<EdgeCase>)`.

    **Helper function** for binary operations:
    ```rust
    fn pop_two(stack: &mut Vec<i64>) -> (i64, i64, Option<EdgeCase>)
    ```
    Pops b (top), then a (second). Returns (a, b, edge_case). If stack has 0 elements: a=0, b=0, edge=EmptyStackDefault. If 1 element: a=0, b=popped, edge=StackUnderflow.

    **Stack operations:**
    - `dup`: peek top (unwrap_or(0)), push copy. Empty → push 0 + EmptyStackDefault.
    - `swap`: if len >= 2, swap last two elements. Else no-op.
    - `drop`: pop top. Empty → no-op.
    - `rotate`: if len >= 3, remove element at `len-3`, push to top (result: `a b c → b c a`). Else no-op.

    **Arithmetic** (all use pop_two + wrapping):
    - `add`: `a.wrapping_add(b)`
    - `sub`: `a.wrapping_sub(b)` (a is second element, b is top — result is a-b per spec)
    - `mul`: `a.wrapping_mul(b)`
    - `div`: if b==0 → push 0, EdgeCase::DivisionByZero. Else `a.wrapping_div(b)`.
    - `mod`: if b==0 → push 0, EdgeCase::ModuloByZero. Else `a.wrapping_rem(b)`.

    **Comparisons** (all use pop_two):
    - `eq`: push `if a == b { 1 } else { 0 }`
    - `gt`: push `if a > b { 1 } else { 0 }`
    - `lt`: push `if a < b { 1 } else { 0 }`

    **Memory:**
    - `store`: pop addr (top), pop val (second) via pop_two. If addr < 0 → no-op (NegativeAddress). Else `memory.insert(addr, val)`. If val is 0, can optionally remove key to keep map sparse.
    - `load`: pop addr (or 0 if empty). If addr < 0 → push 0 (NegativeAddress). Else push `memory.get(&addr).copied().unwrap_or(0)`.

    **I/O:**
    - `print_num`: pop top (or 0 if empty). Format as decimal string `format!("{}", val)`. Extend output_buffer with bytes. Set StepResult.output = Some(formatted_bytes).
    - `print_char`: pop top. If stack was empty → no-op (do NOT print NUL per spec). Else compute `(val % 256) as u8` (handle negative: `((val % 256 + 256) % 256) as u8` or use `val.rem_euclid(256) as u8`). Write byte to output_buffer. Set StepResult.output = Some(vec![byte]).
    - `read`: push one byte from input_buffer.pop_front(), or 0 if empty.

    Update `crates/haydn-vm/src/lib.rs`:
    - Add `mod opcodes;`
    - In `step()`, when processing `Event::Op(opcode)`: match on opcode and call corresponding function from opcodes module. For LoopStart/LoopEnd, still return Noop (Plan 02).
    - Step result must include: `event`, `operation` from opcode function, `stack_snapshot: self.stack.clone()`, `output` from I/O ops, `edge_case` from opcode function.

    Write comprehensive `#[cfg(test)] mod tests` in `opcodes.rs` (or a separate test module) covering every behavior listed above. Use `process_event()` to drive the VM and assert on StepResult fields.

    **Critical spec details to honor:**
    - `sub` operand order: b is top, a is second, result is a-b (NOT b-a)
    - `div` truncates toward zero (Rust default for integers ✓)
    - `mod` sign follows dividend (Rust default for `%` on integers ✓)
    - `print_char` on empty stack is no-op (not print NUL)
    - `print_num` on empty stack prints "0"
    - `load` on empty stack loads from address 0 (address defaults to 0)
    - `store` with fewer than 2 elements: missing operands treated as 0 → store 0 at address 0
    - Wrapping arithmetic: use `wrapping_add`, `wrapping_sub`, `wrapping_mul`, `wrapping_div`, `wrapping_rem`
    - Use `i64::wrapping_div` and `i64::wrapping_rem` to handle i64::MIN / -1 (which would overflow)
