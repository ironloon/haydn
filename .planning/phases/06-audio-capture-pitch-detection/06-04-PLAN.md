    Create `crates/haydn-audio/src/capture.rs` implementing the three-thread architecture (per D-05):

    1. **`pub fn find_audio_input_device(device_name: Option<&str>) -> Result<cpal::Device, AudioError>`**
       - Uses `cpal::default_host()`, enumerates `input_devices()`
       - If `device_name` is Some: find by case-insensitive substring match (same pattern as `haydn-performer/src/main.rs` `find_input_device()`)
       - If `device_name` is None: use `default_input_device()`
       - Return `AudioError::DeviceNotFound` with available device names on failure

    2. **`pub fn list_audio_input_devices() -> Result<Vec<String>, AudioError>`**
       - Returns names of all available input devices (same pattern as `haydn-performer/src/main.rs` `list_audio_devices()` but input-only, returns Vec<String> instead of printing)

    3. **`pub fn start_audio_capture(device: cpal::Device, config: AudioConfig) -> Result<(mpsc::Receiver<AudioMsg>, cpal::Stream), AudioError>`** (per D-14)

       Implementation:
       a. Get device's default input config: `device.default_input_config()`. Extract `sample_rate` and `channels`.
       b. Create ring buffer: `HeapRb::<f32>::new(16384)`, split into `(prod, cons)`.
       c. Create mpsc channel: `mpsc::channel::<AudioMsg>()`.
       d. Build cpal input stream using `device.build_input_stream()`:
          - **Sample format handling:** Match on `config.sample_format()`:
            - `SampleFormat::F32`: callback receives `&[f32]`, extract mono (first channel from each interleaved chunk) and push to ring buffer via `prod.push_slice()` or iterate with `prod.try_push()`.
            - `SampleFormat::I16`: callback receives `&[i16]`, normalize each sample to f32: `sample as f32 / i16::MAX as f32`, then push mono channel to ring buffer.
            - Other formats: return `AudioError::ConfigError` with message about unsupported format.
          - The callback MUST NOT allocate, lock, or block. Only ring buffer push operations. (per D-04)
          - Error callback: log to stderr (eprintln!).
       e. Start stream: `stream.play()`.
       f. Spawn analysis thread (`std::thread::spawn`):
          - Create pitch detector based on `config.algorithm`: if "yin" → `YinDetector::new(config.window_size, config.confidence_threshold)`, if "mcleod" → `McLeodDetector::new(config.window_size, config.confidence_threshold)`.
          - Create `NoteStateMachine::new(&config)`.
          - Pre-allocate `analysis_buffer: Vec<f32>` of size `config.window_size`, filled with zeros.
          - **Main analysis loop:**
            ```rust
            loop {
                let available = cons.occupied_len();
                if available >= config.hop_size {
                    // Shift buffer left by hop_size, fill new samples
                    analysis_buffer.copy_within(config.hop_size.., 0);
                    let fill_start = config.window_size - config.hop_size;
                    cons.pop_slice(&mut analysis_buffer[fill_start..]);

                    // Run pitch detection
                    let pitch = detector.detect(&analysis_buffer, actual_sample_rate);

                    // Run state machine (which internally uses gate and onset)
                    let events = state_machine.process_frame(
                        &analysis_buffer[fill_start..], // just the new hop for RMS
                        pitch.as_ref(),
                    );

                    // Send events
                    for event in events {
                        if tx.send(event).is_err() {
                            return; // Receiver dropped, shut down
                        }
                    }
                } else {
                    std::thread::sleep(std::time::Duration::from_millis(1));
                }
            }
            ```
       g. Return `(rx, stream)`. Caller keeps the stream alive to maintain audio capture.

    4. Add `pub mod capture;` to `crates/haydn-audio/src/lib.rs`. Re-export: `pub use capture::{start_audio_capture, find_audio_input_device, list_audio_input_devices};`.

    5. Verify compilation: `cargo check -p haydn-audio`.

    Create `crates/haydn-audio/tests/integration.rs` — tests that exercise the full pipeline without real audio hardware:

    **Approach:** Since `start_audio_capture()` requires a real cpal device (OS audio hardware), integration tests test the analysis pipeline directly by simulating what the analysis thread does. This is the same pattern as Phase 4 MIDI tests — test the processing pipeline, not the OS I/O layer.

    1. **Helper: `run_pipeline(samples: &[f32], config: &AudioConfig) -> Vec<AudioMsg>`**
       - Creates a pitch detector (McLeod by default)
       - Creates a NoteStateMachine
       - Processes samples in hop-sized chunks through the full pipeline:
         - Fill analysis buffer (same sliding window as capture.rs)
         - Run pitch detection
         - Run state machine
         - Collect all AudioMsg events
       - Returns all events produced

    2. **Helper: `generate_note_sequence(notes: &[(f32, usize)], silence_frames: usize, sample_rate: u32, hop_size: usize) -> Vec<f32>`**
       - Generates a sequence of sine waves at given frequencies for given durations (in frames), with silence gaps between them. Simulates a performance: silence → note → silence → note → ...

    3. **Tests:**

       **test_silence_produces_no_events:**
       - 5 seconds of silence → zero AudioMsg events

       **test_single_note_produces_note_on_and_off:**
       - 0.5s silence → 1.0s of 440Hz sine → 0.5s silence
       - Expect: exactly one NoteOn with note=69 and one NoteOff

       **test_two_notes_sequential:**
       - 0.3s silence → 0.5s of 440Hz → 0.3s silence → 0.5s of 523Hz (C5) → 0.3s silence
       - Expect: NoteOn(69), NoteOff, NoteOn(72), NoteOff (4 events total)

       **test_sustained_note_no_duplicates:**
       - 3.0s of continuous 440Hz sine
       - Expect: exactly one NoteOn(69) (no duplicates)

       **test_note_change_without_silence:**
       - 0.5s of 440Hz immediately followed by 0.5s of 523Hz (no gap)
       - Expect: NoteOn(69), NoteOff, NoteOn(72) (note change detected via pitch change + onset)

       **test_low_frequency_guitar_e2:**
       - 1.0s of 82.41Hz sine (low E guitar string)
       - Expect: NoteOn with note=40

       **test_noise_below_gate:**
       - Random noise scaled to amplitude 0.0001 (well below -40dB gate)
       - Expect: zero events

    4. All tests use `AudioConfig::default()` (window=2048, hop=512, sample_rate=44100).
