<objective>
Implement onset detection, noise gating, and the note state machine that converts frame-by-frame analysis results into discrete NoteOn/NoteOff events.

Purpose: These modules sit between raw pitch detection output and the AudioMsg event stream. They solve the "continuous signal → discrete events" problem: knowing WHAT note is playing (pitch) is half the work; knowing WHEN a new note starts (onset) and filtering noise (gate) completes it.
Output: Working onset detector, noise gate, and note state machine with tests.
</objective>

    1. Create `crates/haydn-audio/src/gate.rs`:
       - **`NoiseGate` struct** with `threshold_db: f32`
       - **`NoiseGate::new(threshold_db: f32) -> Self`**
       - **`pub fn rms_db(samples: &[f32]) -> f32`** — compute RMS: `sqrt(sum(s*s) / len)`, convert to dB: `20.0 * rms.log10()`. If RMS is 0.0, return `f32::NEG_INFINITY`.
       - **`pub fn is_active(&self, samples: &[f32]) -> bool`** — returns `rms_db(samples) > self.threshold_db`. True means "signal is above noise floor" (gate is open).

    2. Create `crates/haydn-audio/src/onset.rs`:
       - **`OnsetDetector` struct** with fields:
         - `threshold_db: f32` — energy spike threshold (from `AudioConfig.onset_threshold_db`)
         - `prev_rms_db: f32` — previous frame's RMS in dB (initialized to `f32::NEG_INFINITY`)
         - `running_avg_db: f32` — exponential moving average of RMS for baseline (alpha = 0.1)
       - **`OnsetDetector::new(threshold_db: f32) -> Self`**
       - **`pub fn process_frame(&mut self, samples: &[f32]) -> OnsetResult`** where `OnsetResult` is:
         ```rust
         pub struct OnsetResult {
             pub is_onset: bool,
             pub current_rms_db: f32,
         }
         ```
         Logic:
         - Compute `current_db = NoiseGate::rms_db(samples)`
         - `delta = current_db - self.running_avg_db`
         - `is_onset = delta > self.threshold_db && current_db > self.prev_rms_db`
         - Update `self.running_avg_db = 0.9 * self.running_avg_db + 0.1 * current_db` (but clamp to not go below -80dB to avoid -inf propagation)
         - Update `self.prev_rms_db = current_db`
         - Return `OnsetResult { is_onset, current_rms_db: current_db }`

    3. Add `pub mod gate; pub mod onset;` to `crates/haydn-audio/src/lib.rs` and add re-exports for `NoiseGate`, `OnsetDetector`, `OnsetResult`.

  <action>
    1. Create `crates/haydn-audio/src/state_machine.rs`:

       **`NoteState` enum:** `Silence`, `Onset`, `Active { note: u8, confidence: f32 }`, `Transition`

       **`NoteStateMachine` struct** with fields:
       - `state: NoteState`
       - `gate: NoiseGate`
       - `onset: OnsetDetector`
       - `config: AudioConfig` (for pitch_stability_cents, min_note_ms, confidence_threshold)
       - `last_note_time_ms: f64` — timestamp of last NoteOn emission (for debounce)
       - `onset_frames: u8` — frames since entering ONSET state (for onset timeout)
       - `sample_rate: u32`
       - `hop_size: usize`

       **`NoteStateMachine::new(config: &AudioConfig) -> Self`** — constructs gate, onset from config values. Initializes state to `Silence`.

       **`pub fn process_frame(&mut self, samples: &[f32], pitch: Option<&PitchEstimate>) -> Vec<AudioMsg>`**

       Returns 0, 1, or 2 AudioMsg events per frame. Logic by state:

       **Silence:**
       - If gate is NOT active → stay Silence, return empty
       - If gate IS active AND onset detected → transition to Onset, reset onset_frames=0, return empty
       - If gate IS active but no onset → stay Silence (soft signal, not a new note), return empty

       **Onset:** (waiting for pitch confirmation)
       - Increment onset_frames
       - If pitch is Some with confidence ≥ config.confidence_threshold:
         - If debounce allows (elapsed since last_note_time_ms ≥ config.min_note_ms):
           - Transition to Active { note, confidence }
           - Update last_note_time_ms
           - Return `[AudioMsg::NoteOn { note, confidence }]`
         - Else: stay in Onset (debounce blocks)
       - If onset_frames > 5 (timeout ~55ms at 512 hop): transition back to Silence, return empty
       - Otherwise: stay in Onset

       **Active { note, confidence }:**
       - If gate is NOT active → transition to Silence, return `[AudioMsg::NoteOff]`
       - If onset detected AND pitch is Some with different midi_note (delta > pitch_stability_cents / 100 semitones, simplified: different midi_note):
         - If debounce allows:
           - Return `[AudioMsg::NoteOff, AudioMsg::NoteOn { note: new_note, confidence: new_confidence }]`
           - Transition to Active { note: new_note, confidence: new_confidence }
           - Update last_note_time_ms
         - Else: stay Active (debounce blocks note change)
       - If pitch is Some with same midi_note → stay Active (sustained note, no event)
       - If pitch is None but gate still active → stay Active (brief pitch dropout during sustain, don't cut note)

       **Transition:** (not used as persistent state — handled inline by Active→Silence and Active→new Active transitions)

       **Helper:** `fn elapsed_ms(&self) -> f64` — compute time elapsed since last_note_time_ms based on frames processed and hop_size/sample_rate.

    2. Add `pub mod state_machine;` to `crates/haydn-audio/src/lib.rs` and re-export `NoteStateMachine`, `NoteState`.

       - **test_silence_no_events:** Feed 10 silence frames → no AudioMsg emitted
