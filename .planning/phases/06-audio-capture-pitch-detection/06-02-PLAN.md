  <action>
    Create `crates/haydn-audio/src/pitch/yin.rs` implementing the YIN algorithm:

    1. **`YinDetector` struct** with fields:
       - `window_size: usize` (typically 2048)
       - `threshold: f32` (typically 0.10–0.20, default 0.15 per research)
       - `diff: Vec<f32>` — pre-allocated buffer for difference function (size = window_size / 2)
       - `cmnd: Vec<f32>` — pre-allocated buffer for cumulative mean normalized difference (size = window_size / 2)

    2. **`YinDetector::new(window_size: usize, threshold: f32) -> Self`** — allocates diff and cmnd buffers.

    3. **`impl PitchDetector for YinDetector`** — `detect()` method implementing:
       - **Step 1 — Difference function:** For each lag τ from 1 to W/2: `d(τ) = Σ_{j=0}^{W/2-1} (x[j] - x[j+τ])²`. Store in `self.diff`.
       - **Step 2 — CMND:** `d'(0) = 1.0`. For τ ≥ 1: `d'(τ) = d(τ) / ((1/τ) * Σ_{j=1}^{τ} d(j))`. If denominator is 0, set d'(τ) = 1.0. Store in `self.cmnd`.
       - **Step 3 — Absolute threshold:** Find first τ ≥ 2 where `cmnd[τ] < threshold`. If none found below threshold, find the global minimum of cmnd (τ ≥ 2). Confidence = `1.0 - cmnd[τ_best]`.
       - **Step 4 — Parabolic interpolation:** If τ_best is not at boundaries (τ > 1 and τ < len-1), fit parabola through cmnd[τ-1], cmnd[τ], cmnd[τ+1] to refine: `refined_τ = τ + 0.5 * (cmnd[τ-1] - cmnd[τ+1]) / (cmnd[τ-1] - 2*cmnd[τ] + cmnd[τ+1])`.
       - **Step 5 — Frequency:** `f0 = sample_rate as f32 / refined_τ`. Validate f0 is in range 20.0..5000.0 Hz, otherwise return None.
       - Use `freq_to_midi(f0)` to get midi_note and cents_offset.
       - Return `Some(PitchEstimate { frequency_hz: f0, midi_note, cents_offset, confidence })` if confidence ≥ 0.3.
       - `name()` returns `"yin"`.

    4. **Early bailout:** Before computing diff, check RMS of input. If RMS < 0.01, return None immediately (silence/noise filter to avoid expensive computation on dead air).

    5. **Add `pub mod yin;`** to `pitch/mod.rs` and re-export: `pub use yin::YinDetector;`.
  </action>

    Create `crates/haydn-audio/src/pitch/mcleod.rs` implementing the McLeod NSDF algorithm:

    1. **`McLeodDetector` struct** with fields:
       - `window_size: usize` (typically 2048)
       - `clarity_threshold: f32` (typically 0.7 — minimum peak height to accept, per D-03 confidence threshold)
       - `nsdf: Vec<f32>` — pre-allocated buffer for NSDF values (size = window_size)

    2. **`McLeodDetector::new(window_size: usize, clarity_threshold: f32) -> Self`** — allocates nsdf buffer.

    3. **`impl PitchDetector for McLeodDetector`** — `detect()` method implementing:
       - **Step 1 — Autocorrelation r(τ):** For each lag τ from 0 to W-1: `r(τ) = Σ_{j=0}^{W-1-τ} x[j] * x[j+τ]`. (Standard time-domain autocorrelation.)
       - **Step 2 — NSDF normalization:** For each lag τ: `m(τ) = Σ_{j=0}^{W-1-τ} (x[j]² + x[j+τ]²)`. Then `nsdf(τ) = 2 * r(τ) / m(τ)`. If `m(τ)` is 0, set nsdf(τ) = 0.
       - **Step 3 — Key maxima selection:** Find positive peaks where nsdf crosses from negative to positive. Track the first such peak above `clarity_threshold`. If none found, take the highest positive peak. Confidence = peak value (already 0.0–1.0).
       - **Step 4 — Parabolic interpolation:** Same as YIN — refine peak position using the 3-point parabolic fit.
       - **Step 5 — Frequency:** `f0 = sample_rate as f32 / refined_τ`. Validate 20.0..5000.0 Hz range.
       - Use `freq_to_midi(f0)` for midi_note and cents_offset.
       - Return `Some(PitchEstimate { ... })` if confidence ≥ 0.3 and peak found.
       - `name()` returns `"mcleod"`.

    4. **Early bailout:** Same RMS < 0.01 check as YIN — skip computation on silence.

    5. **Add `pub mod mcleod;`** to `pitch/mod.rs` and re-export: `pub use mcleod::McLeodDetector;`.
