**Audio quiet mode event loop** (inside the `is_audio_mode` branch of the quiet section):

```rust
while running.load(Ordering::Relaxed) {
    match audio_rx.recv_timeout(Duration::from_millis(500)) {
        Ok(haydn_audio::AudioMsg::NoteOn { note, confidence }) => {
            let velocity = (confidence * 127.0) as u8;  // Per D-14: confidence → velocity
            for line in process_note(note, velocity, &mut engine, &mut vm) {

        }
        Ok(haydn_audio::AudioMsg::SignalLevel(_)) => {
```

**Audio TUI mode event loop** (inside the `is_audio_mode` branch of the TUI section):
Similar to MIDI TUI loop but receives `AudioMsg` and uses audio-specific TUI fields:
```rust
// Create TuiState with audio mode
let mut tui_state = display::TuiState::new(
    engine.metadata().name.clone(),
    connected_name.clone(),
    "Mic".to_string(),
);

// In the drain loop (non-blocking try_recv):
loop {
    match audio_rx.try_recv() {
        Ok(haydn_audio::AudioMsg::NoteOn { note, confidence }) => {
            let velocity = (confidence * 127.0) as u8;
            if let Some(results) = process_note_structured(note, &mut engine, &mut vm) {
                for result in &results {
                    tui_state.update_from_step(note, velocity, result);
                    // Set confidence on the last-pushed history entry
                    if let Some(last) = tui_state.history.last_mut() {
                        last.confidence = Some(confidence);
                    }
                }
            } else {
                tui_state.history.push(display::HistoryEntry {
                    note_name: note_name(note),
                    velocity,
                    confidence: Some(confidence),
                    operation: "(unmapped)".to_string(),
                    output_text: None,
                    edge_case: None,
                });
            }
            tui_state.loop_state = vm.loop_state_name().to_string();
            tui_state.update_stack_and_output(&vm);
            dirty = true;
        }
        Ok(haydn_audio::AudioMsg::NoteOff) => {}
        Ok(haydn_audio::AudioMsg::SignalLevel(level)) => {
            tui_state.signal_level = Some(level);
            dirty = true;
        }
        Err(std::sync::mpsc::TryRecvError::Empty) => break,
        Err(std::sync::mpsc::TryRecvError::Disconnected) => break,
    }
}
```

The keyboard polling ('q' to quit), frame budget rendering, and terminal init/restore are shared between modes — no duplication needed for those.

```rust
// In timeout handler:
match haydn_audio::list_audio_input_devices() {
    Ok(devices) if !devices.iter().any(|d| d == &connected_name) => {
        // Device disconnected
        // Drop the old stream (it's already in _stream variable)
        // In TUI mode: tui_state.connected = false; render;
        eprintln!("⚠ Audio device disconnected. Waiting for device... (Ctrl+C to quit)");
        // Poll for reconnect:
        loop {
            std::thread::sleep(Duration::from_secs(1));
            if !running.load(Ordering::Relaxed) { break; }
            if let Ok(devs) = haydn_audio::list_audio_input_devices() {
                if devs.iter().any(|d| d == &connected_name) {
                    // Reconnect: find device, recreate capture
                    let new_device = haydn_audio::find_audio_input_device(Some(&connected_name))?;
                    let audio_config = engine.audio_section()
                        .map(|s| haydn::audio_config_from_section(s))
                        .unwrap_or_default();
                    let (new_rx, new_stream) = haydn_audio::start_audio_capture(new_device, audio_config)?;
                    // Replace audio_rx and _stream

                }
```

**Note:** Audio reconnect is more complex than MIDI because it requires replacing the entire capture pipeline (new stream + new receiver). The `audio_rx` and `_stream` variables need to be mutable. Use `let mut audio_rx` and `let mut _stream` at creation, then reassign on reconnect. If the scoping makes this difficult, wrap in a struct or use a function that returns the new rx/stream pair.

**crates/haydn-audio/src/capture.rs — Signal level emission:**
In the analysis thread's main loop (inside the `if available >= hop_size` block), after `state_machine.process_frame()` and sending note events, compute and send signal level:
```rust
// After sending note events, compute RMS for signal level
let hop_buf = &analysis_buffer[fill_start..];
let rms = (hop_buf.iter().map(|s| s * s).sum::<f32>() / hop_buf.len() as f32).sqrt();
if tx.send(AudioMsg::SignalLevel(rms)).is_err() {
    return; // Receiver dropped
}
```
This sends one SignalLevel per hop (~11ms at 44100Hz/512 hop), which is ~90 messages/second — negligible overhead for mpsc. The main loop uses the most recent value for the TUI meter.
