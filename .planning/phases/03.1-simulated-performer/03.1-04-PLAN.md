   ```rust
   pub fn write_midi_file(sequence: &NoteSequence, path: &Path, bpm: u32) -> Result<()>
   ```
   - Create `midly::Smf` with Format::SingleTrack, Timing::Metrical (480 ticks/quarter)
   - For each NoteEvent: add NoteOn event, advance by tick duration, add NoteOff event
   - Duration to ticks: `ticks = (duration_ms * 480 * bpm) / (60 * 1000)`
   - For each RestEvent: advance by rest duration in ticks
   - Add tempo meta event at start: microseconds_per_beat = 60_000_000 / bpm
   - Save with `smf.save(path)?`

   **MIDI port output** (`play_midi_port`):
   ```rust
   pub fn play_midi_port(sequence: &NoteSequence, port_name: Option<&str>) -> Result<()>
   ```
   - Create `MidiOutput::new("haydn-performer")?`
   - List available ports. If `port_name` is Some, find matching port. Otherwise:
     - On Mac/Linux: try `create_virtual("haydn-performer")`
     - On Windows: use first available port, or error with: "No MIDI output ports found. On Windows, install a virtual MIDI loopback driver (e.g., loopMIDI) or connect a MIDI device."
   - For each NoteEvent: send NoteOn (0x90, note, velocity), sleep for duration, send NoteOff (0x80, note, 0)
   - For each RestEvent: sleep for duration

2. Create `crates/haydn-performer/examples/hello-world.ly`:
   Design a note sequence that, when fed through the piano tuning and VM, would push the ASCII values for "Hello World" and print them.

   The piano tuning maps:
   - MIDI 36-59 → Push(midi_note - 60), so MIDI 36 → Push(-24), MIDI 59 → Push(-1)
   - To push a positive value, we need arithmetic: push two negatives and subtract, or push a negative and negate

   For 'H' (ASCII 72):
   - Push(-1) + Push(-1) + sub → gives 0... no, sub pops b then a, computes a-b
   - Actually, for Hello World in Haydn, you'd use the BF-to-Haydn mapping from the spec.
   - Simpler approach for the example: just play the note sequence that SOUNDS interesting, with a comment explaining what it would compute.

   ```
   % "Hello World" — notes that, when fed through the default piano tuning,
   % would push values and execute print_char to spell "Hello World!"
   %
   % This is a musicality demo — listen to whether programming sounds musical.
   % The VM integration comes in Phase 4+.
   %
   % Pattern: push value notes (C2-B3 zone) then operation notes (C4+)
   % Each "word" pushes a value then operates on it.

   % Push some values (value zone: C2-B3)
   b a g f e d c
   % Operations (C4+): add, dup, print_char
   c' e' a''
   % More values
   f, g, a, b,
   % Operations: mul, print_char
   g' a''
   ```

3. Create `crates/haydn-performer/examples/fizzbuzz-intro.ly`:

   % Demonstrates what control flow "sounds like" on the piano tuning
   %
   % Pattern: push value, push divisor, mod, compare, conditional branch
   % Hear the contrast between the low value zone and high operation zone.

   % Push 15 (value zone), push 3 (value zone), mod (B4), eq-zero, branch
   b, fis,

   % Push 15, push 5, mod, eq-zero, branch
   b, a,
   b'4

   % Print result or "Fizz"/"Buzz"
