## Architecture Patterns

### Pipeline Architecture

```
LilyPond text (.ly file)
    ↓ [nom parser]
NoteSequence (Vec<NoteEvent>)
    ↓ [SynthBackend trait dispatch]
    ├── BuiltinSynth (layers 0-3: sine → ADSR → waveforms → expressive)
    ├── SoundFontSynth (layer 4: rustysynth + .sf2 file)
    └── MidiBackend (layer 5: midir virtual port OR midly .mid file)
    ↓
rodio Source (f32 stereo stream)
    ↓ [rodio Sink]
Audio output (speakers)
```

### Core Types

```rust
/// A single parsed note from LilyPond
pub struct NoteEvent {
    pub midi_note: u8,       // 0-127, C4 = 60
    pub duration: Duration,  // Computed from tempo + note value
    pub velocity: u8,        // Default 80, modified by dynamics
}

/// Rest events
pub struct RestEvent {
    pub duration: Duration,
}

/// A parsed score
pub enum ScoreEvent {
    Note(NoteEvent),
    Rest(RestEvent),
}

/// The public event stream API (interoperability surface for Phase 4+)
pub type NoteSequence = Vec<ScoreEvent>;
```

### Synthesis Backend Trait

```rust
pub trait SynthBackend {
    /// Synthesize the entire note sequence into a playable audio source
    fn synthesize(&self, sequence: &[ScoreEvent], sample_rate: u32)
        -> Box<dyn rodio::Source<Item = f32> + Send>;

    /// Human-readable name for display
    fn name(&self) -> &str;
}
```

Layers 0-3 implement `SynthBackend` via a `BuiltinSynth` struct parameterized by fidelity level. Layer 4 (`SoundFontSynth`) wraps `rustysynth::Synthesizer`. Layer 5 (`MidiBackend`) doesn't produce audio — it either sends to a virtual port or writes a .mid file.

### rodio Integration Pattern

rodio's `Source` trait requires:
```rust
trait Source: Iterator<Item = f32> {
    fn current_frame_len(&self) -> Option<usize>;
    fn channels(&self) -> u16;         // 2 for stereo
    fn sample_rate(&self) -> u32;      // 44100
    fn total_duration(&self) -> Option<Duration>;
}
```

The synthesizer pre-renders or streams into an `Iterator<Item = f32>` and implements `Source`. rodio handles the rest:

```rust
let (_stream, stream_handle) = rodio::OutputStream::try_default()?;
let sink = rodio::Sink::try_new(&stream_handle)?;
let source = synth_backend.synthesize(&sequence, 44100);
sink.append(source);
sink.sleep_until_end();
```

### Real-Time Rendering with rustysynth

rustysynth renders in blocks. For streaming playback with rodio:

```rust
// Render in blocks (e.g., 512 samples at a time)
let block_size = synth.get_block_size(); // typically 64
let mut left = vec![0f32; block_size];
let mut right = vec![0f32; block_size];

// For each block:
synth.render(&mut left, &mut right);
// Interleave L/R into Iterator<Item = f32> for rodio
```

Schedule `note_on`/`note_off` calls by tracking elapsed samples against note durations:

```rust
synth.note_on(0, midi_note as i32, velocity as i32);
// render N frames worth of samples
synth.note_off(0, midi_note as i32);
```

## LilyPond Subset Specification

### Supported Syntax (per D-02)

| Element | LilyPond Syntax | Example | MIDI Mapping |
|---------|----------------|---------|--------------|
| Note names | `c d e f g a b` | `c` | C in current octave |
| Octave up | `'` | `c''` | +12 per mark from reference |
| Octave down | `,` | `c,,` | -12 per mark from reference |
| Sharp | `is` | `cis` | +1 semitone |
| Flat | `es` | `bes` | -1 semitone |
| Duration | digit | `c4` (quarter), `c8` (eighth) | Affects timing, not pitch |
| Dotted | `.` after duration | `c4.` | 1.5× duration |
| Rest | `r` + duration | `r4` | Silence for duration |
| Barline | `|` | `c4 d e f |` | Ignored (human readability) |
| Tie | `~` | `c4~ c4` | Combine durations |

### Absolute vs Relative Mode

**Absolute mode** (default for this parser): Each note name maps to a fixed octave. `c` = C4 (MIDI 60). Octave marks adjust from there.

**Relative mode** (`\relative c' { ... }`): Each note is relative to the previous. The interval is always less than a fifth — if the literal interval is larger, flip the octave direction. Track `last_note` state.

### nom Parser Structure

```
score       = ws* (event ws*)* EOF
event       = note | rest | barline
note        = pitch duration? dot? tie?
pitch       = note_name accidental? octave*
note_name   = 'c' | 'd' | 'e' | 'f' | 'g' | 'a' | 'b'
accidental  = "is" | "es" | "isis" | "eses"
octave      = '\'' | ','
duration    = '1' | '2' | '4' | '8' | '16' | '32' | '64'
dot         = '.'
tie         = '~'
rest        = 'r' duration? dot?
barline     = '|'
ws          = ' ' | '\n' | '\t' | '%' comment
comment     = '%' (not newline)*
```

The parser produces `Vec<ScoreEvent>` ready for synthesis. Duration state carries forward (if note has no explicit duration, inherit from previous — standard LilyPond behavior).

## Built-in Synthesis (Layers 0-3)

### Layer 0: Pure Sine Wave
```rust
sample = amplitude * (2.0 * PI * frequency * t / sample_rate).sin()
```
~20 lines of code. Test baseline.

### Layer 1: Sine + ADSR Envelope

ADSR state machine:
```
Attack  → amplitude ramps 0.0 → 1.0 over attack_time
Decay   → amplitude ramps 1.0 → sustain_level over decay_time
Sustain → amplitude holds at sustain_level until note_off
Release → amplitude ramps sustain_level → 0.0 over release_time
```

Default ADSR for piano-like: attack=10ms, decay=100ms, sustain=0.7, release=200ms.

### Layer 2: Rich Waveforms

Additive synthesis with harmonic series:
```rust
// Saw approximation (bandlimited to avoid aliasing)
sample = sum(1..N) { (-1)^(k+1) * sin(2π * k * freq * t) / k }
// Triangle
sample = sum(1..N) { (-1)^k * sin(2π * (2k+1) * freq * t) / (2k+1)^2 }
```

Or use wavetable synthesis for better performance — pre-compute one cycle at various pitches.

### Layer 3: Expressive

- **Vibrato**: LFO (5-7 Hz) modulating frequency: `freq * (1.0 + depth * sin(2π * lfo_rate * t))`
- **Tremolo**: LFO modulating amplitude: `amp * (1.0 + depth * sin(2π * lfo_rate * t))`
- **Velocity dynamics**: Scale amplitude by `velocity / 127.0`
- **Legato transitions**: Short crossfade (10-20ms) between consecutive notes instead of hard cut

### Fidelity Flag Mapping

| `--fidelity` | Layer | Description |
|--------------|-------|-------------|
| 0 | Sine only | Beep test mode |
| 1 | Sine + ADSR | Default — notes sound like notes |
| 2 | Rich waveforms + ADSR | Musical character |
| 3 | Expressive + vibrato | Performance-quality |

## MIDI Output (Layer 5)

### Virtual MIDI Port (midir)

```rust
let midi_out = midir::MidiOutput::new("haydn-performer")?;
let ports = midi_out.ports();
// Connect to first available port, or create virtual port
let conn = midi_out.connect(&ports[0], "haydn-out")?;
// Or: midi_out.create_virtual("haydn-performer")  // Mac/Linux only

// Send note-on: status byte + note + velocity
conn.send(&[0x90, midi_note, velocity])?;
// After duration:
conn.send(&[0x80, midi_note, 0])?;
```

**Platform limitation:** Virtual MIDI ports are NOT supported on Windows via midir (WinMM backend doesn't support them). On Windows, user must have a MIDI loopback driver (loopmidi, etc.) or use a real MIDI device. Document this.

## Don't Hand-Roll

| Problem | Use Instead | Why |
|---------|-------------|-----|
| Audio device management | rodio | Cross-platform, handles WASAPI/CoreAudio/ALSA automatically |
| SoundFont loading & synthesis | rustysynth | SF2 spec is thousands of pages. rustysynth handles zones, loops, modulators, filters |
| MIDI port I/O | midir | Platform-specific MIDI APIs are hairy (WinMM, CoreMIDI, ALSA seq) |
| MIDI file format | midly | SMF format has variable-length encoding, multi-track, tempo maps |
| Terminal rendering | ratatui | Alternate screen, raw mode, layout, widgets — don't fight crossterm directly |
| Parser combinators | nom | Error recovery, backtracking, composition — hand-rolled recursive descent works but nom gives you these for free |

**DO hand-roll:**
- LilyPond → MIDI note number conversion (30 lines of pitch math)
- ADSR envelope generator (50 lines state machine)
- Sine/saw/triangle/square waveforms (10-20 lines each)
- Vibrato/tremolo LFOs (10 lines each)
- The `SynthBackend` trait and routing logic

## Common Pitfalls

| Pitfall | How to Avoid |
|---------|--------------|
| **Audio buffer underrun** (clicks/pops) | Use rodio's streaming — don't try to pre-render entire score into memory. Render in chunks via `Source` iterator |
| **LilyPond duration carry-forward** | Track `last_duration` state. If note has no explicit duration, inherit. Forgetting this breaks all but the first note |
| **LilyPond relative mode octave logic** | The "nearest note" rule: if interval > fourth, flip. Must handle this for `\relative` even if absolute mode is default |
| **MIDI note overflow** | Clamp to 0-127 after octave/accidental math. LilyPond `c'''''` would be MIDI 120, but `c''''''` = 132 — overflows u8 |
| **SoundFont not found** | Fail with clear error and download instructions, not a panic. Make `--soundfont` required for layer 4 or warn gracefully |
| **Windows virtual MIDI** | midir doesn't support virtual ports on Windows. Document limitation. Suggest loopmidi for Windows users |
| **rodio OutputStream drop** | `_stream` handle MUST be kept alive for the duration of playback. If dropped, audio stops immediately |
| **Tempo math** | BPM → seconds per beat → seconds per note value. Quarter note at 120 BPM = 0.5s. Eighth = 0.25s. Dotted quarter = 0.75s. Get this wrong and all rhythms are broken |
| **nom v8 breaking changes** | nom 8 changed function signatures vs nom 7. Use `Parser::parse()` trait method, not bare function calls |

## Code Examples

### Minimal nom Parser for Note Names

```rust
use nom::{
    branch::alt,
    bytes::complete::tag,
    character::complete::{char, one_of},
    combinator::{map, opt, recognize},
    multi::many0,
    sequence::tuple,
    IResult, Parser,
};

fn note_name(input: &str) -> IResult<&str, u8> {
    map(one_of("cdefgab"), |c| match c {
        'c' => 0, 'd' => 2, 'e' => 4, 'f' => 5,
        'g' => 7, 'a' => 9, 'b' => 11, _ => unreachable!()
    }).parse(input)
}

fn accidental(input: &str) -> IResult<&str, i8> {
    alt((
        map(tag("isis"), |_| 2i8),
        map(tag("eses"), |_| -2i8),
        map(tag("is"), |_| 1i8),
        map(tag("es"), |_| -1i8),
    )).parse(input)
}

fn octave_marks(input: &str) -> IResult<&str, i8> {
    map(
        many0(alt((
            map(char('\''), |_| 1i8),
            map(char(','), |_| -1i8),
        ))),
        |marks| marks.iter().sum()
    ).parse(input)
}
```

### Minimal rodio Source for Sine Wave

```rust
struct SineSource {
    sample_rate: u32,
    frequency: f32,
    sample_index: u64,
    total_samples: u64,
}

impl Iterator for SineSource {
    type Item = f32;
    fn next(&mut self) -> Option<f32> {
        if self.sample_index >= self.total_samples { return None; }
        let t = self.sample_index as f32 / self.sample_rate as f32;
        let sample = (2.0 * std::f32::consts::PI * self.frequency * t).sin();
        self.sample_index += 1;
        Some(sample)
    }
}

impl rodio::Source for SineSource {
    fn current_frame_len(&self) -> Option<usize> { None }
    fn channels(&self) -> u16 { 1 }
    fn sample_rate(&self) -> u32 { self.sample_rate }
    fn total_duration(&self) -> Option<Duration> {
        Some(Duration::from_secs_f64(
            self.total_samples as f64 / self.sample_rate as f64
        ))
    }
}
```

## Validation Architecture

### Testable Boundaries

1. **Parser correctness**: `parse("c'4 d' e' f' |")` → `[NoteEvent(60, quarter), NoteEvent(62, quarter), ...]`
2. **MIDI note math**: `note_to_midi("c", 0, 4)` → 60, `note_to_midi("fis", 0, 5)` → 78
3. **Duration math**: `duration_from_value(4, false, 120)` → 500ms, `duration_from_value(4, true, 120)` → 750ms (dotted)
4. **Synthesis output**: Non-silent f32 samples of correct length (±1 sample)
5. **MIDI file output**: Written .mid parseable by midly, contains expected note events

### What Can't Be Automated

- Audio quality (subjective) → checkpoint:human-verify after Layer 1
- Musical pleasantness of tuning file sequences → checkpoint:human-verify
