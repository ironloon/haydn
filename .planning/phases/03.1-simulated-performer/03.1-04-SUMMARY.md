# Plan 03.1-04 Summary: MIDI Output, TUI Display, Examples

## What Was Done

### Task 1: MIDI Output Backend (Layer 5)
- Added `midly = "0.5"` and `midir = "0.10"` dependencies
- Created `midi.rs` with two modes:
  - **File export** (`write_midi_file`): Writes Standard MIDI File (Format 0, 480 ticks/quarter) with tempo meta event, note-on/off events, and proper timing
  - **Port output** (`play_midi_port`): Real-time MIDI via midir — auto-detects ports, sends note-on/note-off with sleep timing
- Added CLI flags: `--midi-out` (file path) and `--midi-port` (named port)
- Wired into main.rs: `--synth midi --midi-out path.mid` writes file, `--synth midi` sends to port

### Task 2: TUI Playback Display
- Added `ratatui = "0.29"` and `crossterm = "0.28"` dependencies
- Created `display.rs` with music player UX:
  - Scrolling note display with current note highlighted: `C4 → [D4] → E4`
  - Progress bar (Gauge widget) with percentage
  - Measure and note counters
  - Backend/BPM info line
  - 'q' to quit, proper terminal cleanup
- Integrated into main.rs: TUI runs in a separate thread alongside audio playback
- `--quiet` mode skips TUI entirely

### Task 3: Example LilyPond Scores
- Updated `examples/scale.ly` — ascending/descending C major scale
- Created `examples/hello-world.ly` — value+operation zone patterns demonstrating programming sounds
- Created `examples/fizzbuzz-intro.ly` — control flow patterns (push, mod, compare, branch)
- All three parse and play successfully
