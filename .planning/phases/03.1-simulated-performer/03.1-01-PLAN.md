Implement `crates/haydn-performer/src/parser.rs`:

1. Define a `ParseError` enum with thiserror for parser errors (invalid note name, invalid duration, unexpected character).

2. Define internal parser types:
   ```rust
   struct ParseState {
       default_duration: u32,  // LilyPond duration value (4=quarter, 8=eighth)
       bpm: u32,               // Tempo for duration calculation
   }
   ```

3. Implement nom parsers for each LilyPond element (per RESEARCH.md grammar):
   - `note_name(input) -> IResult<&str, u8>`: Maps c=0, d=2, e=4, f=5, g=7, a=9, b=11
   - `accidental(input) -> IResult<&str, i8>`: "is"=+1, "es"=-1, "isis"=+2, "eses"=-2
   - `octave_marks(input) -> IResult<&str, i8>`: Count ' (+1 each) and , (-1 each)
   - `duration_value(input) -> IResult<&str, u32>`: 1, 2, 4, 8, 16, 32, 64
   - `dot(input) -> IResult<&str, bool>`: Optional '.'
   - `tie(input) -> IResult<&str, bool>`: Optional '~'
   - `rest(input) -> IResult<&str, ScoreEvent>`: 'r' + optional duration + dot
   - `note(input) -> IResult<&str, PartialNote>`: note_name + accidental? + octave* + duration? + dot? + tie?
   - `barline(input) -> IResult<&str, ()>`: '|'
   - `comment(input) -> IResult<&str, ()>`: '%' until newline
   - `score(input) -> IResult<&str, Vec<ScoreEvent>>`: Top-level parser

4. MIDI note number calculation:
   - Base octave: unmodified note name is in octave 4 when one `'` is present (this is standard LilyPond absolute mode)
   - Actually use LilyPond absolute convention: `c` (no marks) = C3 (MIDI 48), `c'` = C4 (MIDI 60), `c''` = C5 (MIDI 72), `c,` = C2 (MIDI 36)
   - Formula: `midi = 48 + semitone_offset + accidental + (octave_marks * 12)`
   - Where semitone_offset is from note_name (c=0, d=2, e=4, f=5, g=7, a=9, b=11)
   - Clamp result to 0-127

5. Duration calculation:
   - `duration_to_ms(value, dotted, bpm)`: quarter at 120bpm = 500ms, dotted = 1.5×
   - Duration carry-forward: if a note has no explicit duration, use the previous note's duration value

6. Tie handling: when a note has `~`, combine its duration with the next note of the same pitch. Accumulate tied durations during post-processing.

7. Public API:
   ```rust
   pub fn parse(input: &str, bpm: u32) -> Result<NoteSequence, ParseError>
   ```

8. Write unit tests in the same file for each parser combinator.

Create `crates/haydn-performer/tests/parser.rs` with integration tests:

   - Test edge cases: empty input, only whitespace, only comments, only barlines
