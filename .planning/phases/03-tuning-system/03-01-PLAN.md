    Also implement `parse_opcode(name: &str) -> Result<haydn_vm::Opcode, TuningError>`:
    - Match lowercase-trimmed input against all 19 opcode names: "dup", "swap", "drop", "rotate", "add", "sub", "mul", "div", "mod", "eq", "gt", "lt", "loop_start", "loop_end", "print_num", "print_char", "read", "store", "load"

    **Step 4: Create crates/haydn-tuning/src/error.rs** — Error types
    ```rust
    #[derive(Debug, thiserror::Error)]
    pub enum TuningError {
        #[error("TOML parse error: {0}")]
        Parse(#[from] toml::de::Error),

        #[error("in [{section}]: {message}")]
        Validation { section: String, message: String },

        #[error("unknown opcode '{found}'{}", suggestion.as_ref().map(|s| format!(" — did you mean '{s}'?")).unwrap_or_default())]
        UnknownOpcode { found: String, suggestion: Option<String> },

        #[error("overlapping rules: {rule_a} and {rule_b} both match note {note}")]
        Overlap { rule_a: String, rule_b: String, note: u8 },

        #[error("note {note} out of MIDI range (0-127)")]
        NoteOutOfRange { note: u8 },

        #[error("{0}")]
        Io(#[from] std::io::Error),
    }

    pub struct ValidationResult {
        pub errors: Vec<TuningError>,
        pub warnings: Vec<String>,
    }
    ```

  <action>
    **Step 1: Create crates/haydn-tuning/src/validate.rs**

    Implement `pub fn validate(raw: &RawTuningFile) -> ValidationResult`:

    1. **Metadata checks:** `root_note` must be 0-127. `name` must be non-empty.
    2. **Values section (if present):** `range[0] <= range[1]`, both values 0-127.
    3. **Operations table (if present):** Each key must parse as u8 in 0-127. Each value must be a valid opcode name (use `parse_opcode`). Collect errors for ALL invalid entries, don't stop at first.
    4. **Raw rules:** Each trigger validated — note values 0-127, range from <= to, interval semitones in -127..127. Each emit validated — opcode names valid.
    5. **Overlap detection (per D-10):** Expand all rules to the set of MIDI notes they match:
       - ExactNote(n) → {n}
       - NoteRange(from, to) → {from..=to}
       - ValuesSection range → {from..=to} (treated as implicit NoteRange)
       - Operations table entries → {each key}
       - Interval triggers are excluded (they match note *pairs*, not single notes)
       Build a `HashMap<u8, Vec<String>>` mapping each note to rule descriptions. Any note with >1 rule → Overlap error for each pair.
    6. **Coverage check (per D-07, D-09):** Collect the set of all opcodes mapped (from operations table + Op emit rules). If any of the 19 opcodes are unmapped AND `partial_coverage` is false → add warning per missing opcode. If `partial_coverage` is true → skip.

    Return `ValidationResult { errors, warnings }`.

    **Step 2: Wire into lib.rs**
    Add `pub mod validate;` to lib.rs. Add `pub use validate::validate;`.
