TOML Schema (from Plan 01 types, per 03-CONTEXT.md D-04 through D-06):

[metadata]            — REQUIRED
  name: String        — tuning name
  root_note: u8       — MIDI note number (0-127), semitone offsets calculated from this
  instrument: String  — instrument identifier
  author: String      — OPTIONAL
  partial_coverage: bool — OPTIONAL, default false. If true, suppresses missing-opcode warnings.

[values]              — OPTIONAL shorthand
  range: [u8, u8]     — inclusive MIDI note range, produces Push(semitone_offset_from_root)

[operations]          — OPTIONAL shorthand
  {midi_note} = "{opcode_name}"   — maps exact note to opcode

[[rules]]             — OPTIONAL, array of tables
  [rules.trigger]
    type = "note" | "range" | "interval"
    note: u8          — for type "note"
    from: u8, to: u8  — for type "range"
    semitones: i8     — for type "interval"
  [rules.emit]
    type = "push" | "push_offset" | "op"
    value: i64        — for type "push"
    opcode: String    — for type "op"

Evaluation order: values range → operations (sorted by note) → raw rules (array order)
First match wins.

All 19 opcodes: dup, swap, drop, rotate, add, sub, mul, div, mod, eq, gt, lt, loop_start, loop_end, print_num, print_char, read, store, load

    **§8: Validation Errors** (~30 lines)
    Catalog of error messages with explanations and fixes:

    |-------|-------|-----|
    | "unknown opcode 'dupe' — did you mean 'dup'?" | Typo in opcode name | Check spelling against opcode reference |
    | "overlapping rules: ... both match note 60" | Two rules trigger on same note | Remove one rule or change note assignments |

    | "in [values]: range[0] must be <= range[1]" | Inverted range bounds | Swap the two numbers |

    **§9: Tips** (~10 lines)
    - Keep the value zone at least 2 octaves wide for useful numeric range
    - Place frequently-used opcodes on comfortable keys
