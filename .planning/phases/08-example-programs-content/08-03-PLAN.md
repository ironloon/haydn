**Structure** (following esolang wiki conventions):

1. **Language name + brief description** — One paragraph. Haydn is an esoteric programming language where programs are performed live on musical instruments. Stack-based VM, 19 opcodes, event-driven.

2. **Overview** — The key innovation: no source code files. The performer's live musical input IS the program. Two event types: Push(value) and Op(opcode). Tuning files map musical gestures to VM events — different instruments produce different programs. Mention the designer and year.

3. **Computational Model** — Condensed from spec §2:
   - Stack: unbounded LIFO of i64 values
   - Memory: addressable i64 array, non-negative indices
   - I/O: byte streams, non-blocking read
   - Control flow: loop pedal model (while-loop semantics)

4. **Instruction Set** — Full opcode table (condensed from spec §3.3). All 19 opcodes with stack effects and brief descriptions. Use the standard wiki table format:
   ```
   | Opcode | Stack Effect | Description |
   ```
   No need to repeat edge cases — link to the spec for details.

5. **Event Interface** — Brief: two event types, Push(value: i64) and Op(opcode). Source-independent. The VM doesn't know about music.

6. **Control Flow** — The loop pedal model. loop_start peeks (enter if nonzero, skip if zero), loop_end pops (replay if nonzero, exit if zero). Execute-during-record. This gives while-loop semantics.

7. **Hello World** — Pull the event sequence from the spec (§8.1) and also show the piano tuning note-name version from `examples/hello-world.md`. This is what readers want to see first.

8. **Computational Class** — Turing-complete. Proven via reduction from Brainfuck (spec §7). Brief summary: BF tape simulated with Haydn memory, BF instructions compile to fixed Haydn event sequences. Link to the full proof in the spec.

9. **Tuning System** — Brief explanation: TOML config files map MIDI notes/audio pitches to VM events. Default tunings: piano, voice, guitar. Users can create custom tunings.

10. **External Resources** — Links (as placeholders for now):
    - GitHub repository

    - `cargo install haydn`
