**Hello World (`examples/hello-world.md`)** — per D-01 (note names lead), D-03 (full tabular walkthrough), D-09 (teaches push/operate/print):

The program prints "Hello, World!\n" using the default piano tuning. Each ASCII character is built via value-zone pushes and arithmetic. Structure:

2. Piano tuning quick reference (value zone = low notes, operation zone = high notes)
3. Full program in note-name notation, grouped by character output

6. Performance instructions ("Play these notes on any MIDI keyboard with the default piano tuning")

**Arithmetic approach for building ASCII values from Push(-24...-1):**
- H (72): Push(-8) × Push(-9) = 72 → E3, D#3, G4, A5
- e (101): Push(-10) × Push(-10) = 100, then Push(-1), Sub → 100-(-1)=101 → D3, D3, G4, B3, D4, A5
- l (108): Push(-12) × Push(-9) = 108 → C3, D#3, G4, A5
- o (111): Push(-12) × Push(-10) = 120, Push(-9), Add → 111 → C3, D3, G4, D#3, C4, A5
- , (44): Push(-4) × Push(-11) = 44 → G#3, C#3, G4, A5
- (space 32): Push(-4) × Push(-8) = 32 → G#3, E3, G4, A5
- W (87): Push(-9) × Push(-10) = 90, Push(-3), Add → 87 → D#3, D3, G4, A3, C4, A5
- r (114): Push(-6) × Push(-19) = 114 → F#3, F2, G4, A5
- d (100): Push(-10) × Push(-10) = 100 → D3, D3, G4, A5
- ! (33): Push(-3) × Push(-11) = 33 → A3, C#3, G4, A5
- \n (10): Push(-2) × Push(-5) = 10 → A#3, G3, G4, A5

Verify each arithmetic sequence is correct. The executor MUST trace through the VM semantics to confirm (operand ordering: `a b sub` = a - b where b is top of stack).

**Cat (`examples/cat.md`)** — per D-09 (teaches I/O), D-10 (less scaffolding than Hello World):

Cat reads input bytes and echoes them until EOF (read returns 0). Event sequence:
```
Op(Read), Op(Dup), Op(LoopStart),
  Op(Drop), Op(PrintChar), Op(Read), Op(Dup),
Op(LoopEnd), Op(Drop)
```
Trace: read→[b], dup→[b,b], loop_start peeks b (nonzero→enter), drop→[b], print_char→[], read→[next], dup→[next,next], loop_end pops (nonzero→replay, zero→exit), drop cleans up.

Piano notes: C6, E4, C5, F4, A5, C6, E4, D5, F4

Format: Title, what it teaches, note sequence, brief explanation (NO full table — per D-10).

**Truth Machine (`examples/truth-machine.md`)** — per D-09 (teaches conditional loops):

Reads one byte of input. If '0' (ASCII 48): prints "0" and halts. If '1' (ASCII 49): prints "1" forever.

Event sequence (direct Haydn, not BF reduction — cleaner for this program):
```
Op(Read),                              // read input byte
Push(X), Push(Y), Op(Mul),            // build 48 on stack (e.g. -6 × -8)
Op(Sub),                               // input - 48 → 0 or 1
Op(Dup),                               // [v, v]
Push(X), Push(Y), Op(Mul),            // build 48 again
Op(Add), Op(PrintChar),               // v + 48 → ASCII digit, print it. Stack: [v]
Op(LoopStart),                         // peek v: 0→skip, 1→enter
  Op(Dup),                             // [1, 1]
  Push(X), Push(Y), Op(Mul),          // build 49
  Push(Z), Op(Sub),                    // adjust to 49
  Op(PrintChar),                       // print '1'. Stack: [1]
  Op(Dup),                             // [1, 1]
Op(LoopEnd)                            // pop 1→replay forever
```

The executor should work out the exact Push sequences for constants 48 and 49 using the value zone and verify by tracing.
- 48 = (-6) × (-8): F#3, E3, G4
- 49 = (-7) × (-7): F3, F3, G4

**Note sequence (approximate):** C6, F#3, E3, G4, D4, E4, F#3, E3, G4, C4, A5, C5, E4, F3, F3, G4, A5, E4, D5

Format: Title, what it teaches (conditionals, infinite loops), note sequence, explanation of the branching logic.

2. Explain the mechanical BF→Haydn compilation from the spec
3. Show the first few "measures" in note-name form (enough to see the pattern of low→high register alternation)
4. Note that the full event sequence is verified by the integration test

A known compact BF FizzBuzz (prints 1-100 with Fizz/Buzz/FizzBuzz):
The executor should find or write a working BF FizzBuzz. There are well-known implementations ~350-400 chars. The key insight: each BF instruction maps to 4-8 Haydn events via `bf_compile()`, so the full program is ~2000+ events — far too long to notate entirely, but the test proves it works.

Alternatively, the executor can write a **simpler FizzBuzz variant** that counts from 1 to a smaller number (e.g., 1-15 to show one full cycle), using a purpose-built BF program or direct Haydn event construction. Format: Title, what it teaches, BF source + explanation of reduction, opening measures in note-name form, test verification note.
